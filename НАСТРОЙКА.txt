ГРАФИК СМЕН — ЗАЛИТЬ НА GITHUB
================================

Сайт будет тут:
https://vaflya67.github.io/grafik/

═══════════════════════════════════════
ЕСЛИ DEPLOY ПАДАЕТ (красный deploy)
═══════════════════════════════════════

Это НЕ баг календаря. GitHub не может выложить сайт.
Сборка (build) зелёная — файлы норм.

ШАГ 0 — ГЛАВНОЕ (90% случаев)
------------------------------
1. Репо grafik → Settings → Actions → General
2. Прокрути до "Workflow permissions"
3. Выбери: "Read and write permissions"
4. Save

Без этого deploy пишет: "Deployment failed, try again later"

ШАГ 1 — Pages через GitHub Actions (надёжнее)
----------------------------------------------
1. Settings → Pages
2. Build and deployment → Source: "GitHub Actions"
   (НЕ "Deploy from a branch"!)
3. Save

ШАГ 2 — Залить workflow
-----------------------
На GitHub в репо grafik:
1. Add file → Create new file
2. Имя: .github/workflows/pages.yml
   (GitHub сам создаст папки)
3. Скопируй содержимое из папки grafik-smen на рабочем столе
4. Commit

Или перетащи всю папку .github при upload files.

ШАГ 3 — Залить файлы сайта
--------------------------
В корне репо должны быть:
- index.html
- app.js
- style.css
- .nojekyll  (пустой файл, важно!)
- .github/workflows/pages.yml

ШАГ 4 — Проверить Actions
-------------------------
Actions → "Deploy to GitHub Pages" → должно быть зелёное
Через 1–2 мин: https://vaflya67.github.io/grafik/

═══════════════════════════════════════
ПЕРВАЯ ЗАЛИВКА С НУЛЯ
═══════════════════════════════════════

1. github.com → New repository → grafik → Public
2. Upload files (все из папки grafik-smen)
3. Settings → Actions → Read and write permissions
4. Settings → Pages → Source: GitHub Actions
5. Открыть сайт по ссылке выше

На телефоне: «Добавить на главный экран».
